# Agent Session Kit v2.0.0 Announcement Copy

## Short Post

Agent Session Kit `v2.0.0` is live.

This is the ASK 2.0 release and marks a major shift: ASK is now a Developer-Agent Runtime.

Highlights:

- standalone `ask-core` runtime as the governance source of truth
- runtime-only `pre-commit` and `pre-push` policy checks
- stall recovery with `180s` timeout and one automatic retry
- Codex context-budget commands for long-session resilience

Result: safer, more resumable developer-agent workflows with one runtime path from local session state to git guardrails.

Release notes: `docs/releases/v2.0.0.md`

## Blog Intro

`v2.0.0` is a major architectural release for ASK. The project now centers on `ask-core`, a platform-agnostic runtime that governs session lifecycle, context verification, policy checks, and handoff continuity. This cutover replaces legacy split-path verification with a single runtime execution path and adds resilience features such as guarded command retries and Codex context-budget controls.

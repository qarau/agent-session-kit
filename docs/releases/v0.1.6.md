# Agent Session Kit v0.1.6

Release date: 2026-03-11

## Summary

Efficiency Workflow Upgrade (Resume + Archive + Fast-Check).

This release improves day-to-day execution speed and session continuity for ASK-driven development.

## Highlights

### 1) Fast Resume Snapshot

Added:

- `scripts/session/resumeSession.mjs`

What it does:

- Prints current branch and HEAD.
- Reads active objective from `docs/session/current-status.md`.
- Reads next unchecked task from `docs/session/tasks.md`.
- Shows latest verification command from `docs/session/change-log.md`.

Why it matters:

- Faster rehydrate after compaction/interruption.
- Lower resume friction for both humans and agents.

### 2) Session Change-Log Archiver

Added:

- `scripts/session/archiveSessionLog.mjs`

What it does:

- Keeps recent sections in `docs/session/change-log.md`.
- Moves older dated sections into monthly archives:
  - `docs/session/archive/change-log-YYYY-MM.md`

Why it matters:

- Keeps active session files small and readable.
- Preserves full history without bloating working context.

### 3) Fast-Check Lane for V2

Added:

- `apps/gumatua-v2/scripts/fastCheck.mjs`
- `apps/gumatua-v2` `package.json` -> `fast-check`

What it does:

- Runs type-check first.
- Runs targeted tests based on changed-file scope.
- Falls back to full suite for config-level changes.

Why it matters:

- Faster local feedback loops.
- Better default workflow before full CI gate.

### 4) Guardrail Policy Clarification

Updated docs to make strictness behavior explicit:

- `strictTasksDoc` default remains soft (`false`).
- Strict mode remains opt-in via config/env.

# Agent Session Kit v2.0.0

Release date: 2026-03-12

## Summary

ASK 2.0 introduces a major architectural cutover from script-first guardrails to a standalone runtime core.

This release positions ASK as a Developer-Agent Runtime: a platform-agnostic layer that governs developer-agent work in progress through session, context, policy, and handoff controls.

## Highlights

### 1) Standalone `ask-core` Runtime

Added and promoted:

- `ask-core/` as the runtime package and CLI source of truth

What it does:

- Centralizes session lifecycle and work-context verification.
- Centralizes policy enforcement for pre-commit and pre-push checks.
- Provides a stable runtime contract for adapters and hooks.

Why it matters:

- Removes duplicated logic from legacy script paths.
- Makes ASK easier to evolve, test, and port across environments.

### 2) Hard Cutover to Runtime-Only Hook Flow

Changed:

- `pre-commit` now routes through `ask pre-commit-check`.
- `pre-push` now routes through `ask pre-push-check`.
- Legacy `kit/scripts/session/*` verification scripts were removed.

Why it matters:

- One policy engine and one runtime path for commit/push governance.
- Lower risk of drift between wrapper scripts and enforcement logic.

### 3) Runtime Stall Recovery

Added:

- `GuardedCommandRunner` in `ask-core`

What it does:

- Detects stalled adapter command execution.
- Uses `180s` no-output/wall timeout.
- Performs one automatic retry before final failure.

Why it matters:

- Better resilience during long-running or unstable local agent sessions.
- Cleaner failure mode when a run cannot recover.

### 4) Codex Context-Budget Integration

Added:

- `ask codex context status`
- `ask codex context ensure`
- `ask codex context compact`

What it does:

- Tracks remaining context budget for Codex Responses API workflows.
- Can trigger compact + rehydrate flow when below policy threshold.
- Surfaces codex context summary in `ask session doctor`.

Why it matters:

- Reduces mid-task context failures in long sessions.
- Makes context-budget behavior observable and policy-driven.

## Upgrade Notes

- This is a major-version release because runtime governance is now `ask-core`-first.
- Existing installs should re-run installer/hook setup so pre-commit and pre-push point to runtime adapters.
- Teams relying on removed legacy verifier scripts should migrate to `ask` runtime commands and adapters.
